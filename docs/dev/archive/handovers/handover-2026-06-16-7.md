## ⛔ THE GOVERNING PRINCIPLES this session locked (cascade-engine-430.md §2b)

  structure to match the current mod. Two guardrails: **preserve SAVES** (soft format / `@SAVEBREAK` only if unavoidable) +
  **preserve HOW THE GAME WORKS** (intended gameplay; the cascade may correct buggy old *outputs*, not the intended design).
- **Build the 4 components FROM SCRATCH; actively AVOID reusing existing engine code** (NOT `CvInfoUtil`/the old `read()`).
  External libs OK — **`picojson`** (proven, backs `CvHttpServer`). The ONLY kept things = the hard EXE boundary (the type
  registry + the `getNum*/get*Info` accessor surface) + **`BoolExpr`** (the one deliberate reuse — see below).
- **`BoolExpr` is the reuse exception for CONDITIONALS.** The JSON conditionals (`requires`/`enabled`/`disabled`:
  `all`/`any`/`noneOf` over atoms+predicates) are **isomorphic to `BoolExpr`** (`And`/`Or`/`Not` over `Has`(GOM)/`Is`(tag),
  `Sources/BoolExpr.{h,cpp}`). The runtime readJson translates a JSON conditional → a `BoolExpr` tree; the harness proves the
  same parse by rendering it to clear text. One conditional shape, two back-ends (BoolExpr in-game, text offline).
- **readJson is a CONSUMER of the shape, not a producer** — one shape (data-model-spec), consumed by readJson, read by the 3 machines.

## THE TAXONOMY — SETTLED THIS SESSION (step 1; data-model-spec §1)
Owner decisions, locked into §1 + the harness (flags dropped **3098 → 326**):
- **(A) TEXT fields → `identity`** — `description`/`help`/`civilopedia`/`message`/`quote`/`strategy`/`adjective`/`shortDescription`
  nest under `identity` (text IS "what am I").
- **(B) team `capabilities` vs unit `skills` — scope by SECTION NAME, never checked.** A civ has **`capabilities`** (TEAM,
  tech-unlocked: found-on-peaks/pass-peaks/move-on-water/tech-trading/irrigation/bridge-building/river-trade); a unit has
  **`skills`** (UNIT, innate: blitz/walk-on-mountains/fly-over-water-helicopter/amphibious/can-move-impassable/…). DIFFERENT
  mechanics; the parser never infers scope. (Today's tech bools → `CvTeam` counter via `processTech`; unit innate = the
  unit-plane boolean group, rename `capabilities`→`skills`.)
- **(C) bespoke sections ENUMERATED** (object-valued but NOT scope-keyed families): `promotionLine` `buildUp` `shrine`
  `properties` `voteSource` `threshold` `role` `victory` `targetLevel` `conversion` `cityFounding` `unitCapability`.
- **Structural rule:** a non-reserved top-level key is a **modifier family iff its value is an OBJECT** (scope-keyed); a
  **bare value** (bool/string/number) is a capability/skill/text flag, never a family.
- **Palace-type rule (`15a375ee`):** 9 government-center buildings (`bCapital` Palace + 8 `bGovernmentCenter` pseudo-palaces —
  Forbidden Palace/El Escorial/Versailles/Edinburgh Castle/Alhambra/Abu Simbel/Apadana/House of Parliament) carry

  forced capital-move (capital falls → hard-move to the Palace) BYPASSES it (#437 placement-gate invariant). `IS_CAPITAL` =
  "city has a palace/government-center building" (engine computes it from `identity.{capital,governmentCenter}` presence).
